■ vw
viewport width
cssで使用する単位のひとつ。画面の横幅を100として計算する。
例えば、10vwと指定した時画面幅が1200pxだったら、指定した要素は120pxになるし、320pxで閲覧すれば32pxになる。基本的には％指定と同じであるが！
% の場合は対象となる要素のプロパティが親要素のそれと紐付けられるため、必ずしもビューポートの幅が基準になるとは限らない。
例えば親要素の横幅を飛び出して子要素を配置したい
とかそういう時に便利
vwはスクロールバーを含んだサイズになるので、%で指定した場合より少し大きく表示されるので、横スクもでる。

CSS には vw, vh, vmin, vmax という単位がある

■ vh
viewport height
ビューポートの高さに対する割合。
vw同様に画面の高さを100として計算する。
横幅同様に、親要素に関係なく、画面幅に対して指定ができるのでSPのときに便利。

CSSのvwやvhを使ってみる

■ vmin
Viewport Minimum
画面の横幅と高さでで小さいほうを100として計算する。
なので、PCだったら基本的に画面幅より、画面の高さのほうが小さいので、画面の高さを100として計算される。
SPの場合は、画面幅より画面の高さのほうが大きいので、画面幅を100として計算される。
SPを横にした場合は画面幅より画面の高さのほうが小さくなるので、逆に高さを100として計算される。

■ vmax
Viewport Maximum
画面の横幅と高さでで大きいほうを100として計算する。
なので、PCだったら基本的に画面幅を100として計算される。
SPの場合は、画面幅より画面の高さのほうが大きいので、画面の高さを100として計算される。
SPを横にした場合はvminと逆のことが起こる。

CSSのvwやvhを使ってみる

ともろもろ試してみたが使いこなせ無さそう。ネットではfont-size指定に便利とあった。
画面幅によって改行位置がことなったり、見出しから飛び出したり
というのを下の「vw.vh」を使うと解決できるらしい。
たとえば、font-sizeを8vwに設定すると、viewportの幅1200pxに対しては約96px、
400pxに対しては33px、1920pxに対しては154pxになります。
