input要素

★id属性★
どのフォームと関連付けるかを <input>のid名で指定。
つまるところlabelタグのfor属性とinputタグのid属性が同じなら、labelタグ部分をクリックしてもinputタグがアクティブになったりする。（分かりずらい。）
labelタグのfor属性とinputタグのid属性に同じものが設定されてれば問題ないという理解でOK。
★name属性★
name属性は主にシステム周りでつかう属性。システムテンプレート作成は空で作成しても問題ない
inputタグに入力されたデータに名前をつける。
例えば、住所の入力欄にname="address"みたいな。
input typeタグで、idとnameは必ずしも同じ名前にしなければならないわけではない。（下記参考）
  • input typeタグで、idとnameは同じ名前にしなければいけないの？
★value属性★
初期値を設定できる。
ただinput type="text"には設定しなくても問題ない。
radioボタン、checkboxを使うときは設定は必須。
  • value 属性
★disabled属性★
HTML5で追加された属性
input要素のdisabled属性は、input要素を無効にする属性。この指定があると入力などができない。
★required属性★
HTML5で追加された属性
input要素のrequired属性を設定すると、入力必須の項目になる。
これを指定しているinputボタンに何も入力しないで送信しようとすると、ブラウザで勝手にエラー文言がでてくる。（下の、「市区郡」のとこに設定してみた。）
★placeholder属性★
input要素のplaceholder属性で指定した値が、入力欄に初期値として表示される。
select要素にはまだない。。。

input type="file"のAndorid4系でのバグについて
inputタグにスタイルを当てるためにdisplay:none;をかけてるとAndroid4.4でボタンをタップしてもファイル選択ができない。
なので、inputはdisplay:none;ではなく、positionとかでどっかにすっとばして、labelタグでボタンを作るのが正解かも。
（端末によってはこんなことしなくてもファイル選択ができたりするのでAndroid4系が対象端末に含まれている場合は要注意。）

  • Android標準ブラウザでinput[type="file"]を扱う
  • もうすぐクリスマスなのでinputとlabelの仲を引き裂いてみた

}

input type="number"について
type属性を"number"にすると、PCの時、半角数字しか入力されない。また、inputタグホバー時やアクティブ時に右側に数を増減させる矢印（スピンボタン）がでる。
これを非表示にするには、下記参照。
SPのとき、inputをタップすると数字入力のキーボードが表示される。

/*-- Chrome、Safari --/
input[type="number"]::-webkit-outer-spin-button,
input[type="number"]::-webkit-inner-spin-button {
-webkit-appearance: none;
margin: 0;
}
/*-- Firefox、IE --/
input[type="number"] {
-moz-appearance:textfield;
}

optgroupについて
selectタグの中にoptgroupを設定するとiPhoneで見たときプルダウンのメニューが左寄りになってしまう。
センター寄せかつグループ化したいときはoptionタグに「disabled」属性で対応するのが無難そう。 

<select>
<option value="" disabled>関東地方</option>
</select>

IEでの表示について
IEでは"appearance"で右側の矢印を非表示にできないので、下記の記述を追加すること。自前のリセットに追加すべき! 

select::-ms-expand {
display: none;
}

input type="text"のAndorid標準ブラウザでのバグについて
inputタグから入力したテキストがはみ出した時、普通なら横に改行されずにつながっていくが、Android標準ブラウザだと、ある一定数を超すか、inputから飛び出るかすると改行されてinputタグからはみ出して表示されるバグを発見。
端末によっては、そのまま文字が下に飛び出したり、一見問題無さそうに見えても、テキストが見えないだけで、実ははみ出しているので、やけにinputタグの下に余白が出来たりなど、起こったことは少しづつちがうけど、結局inputタグから文字が飛び出していた。
対処法としてinputタグに「word-break: normal;」をかけると上手くいった。。

css4（!?）で追加された疑似クラス。
